# QFeeder Bot — 项目架构

## 概述
Feeding Agent: 自动化微信公众号+小红书量化文章分析系统。

## 目录结构
```
feeding_agent/
├── run.py              — 主CLI入口（subscribe/process/watch/list/xhs-*）
├── config.py           — 配置（API keys、数据路径）
├── telegram_bot.py     — Telegram bot（Claude Sonnet agent）
├── dingtalk_bot.py     — 钉钉机器人
├── core/
│   ├── fetcher.py      — 从 we-mp-rss(localhost:8001) 抓取微信文章
│   ├── xhs_fetcher.py  — 小红书笔记抓取（用户订阅+关键词搜索）
│   ├── parser.py       — HTML解析（文本、代码块、论文链接）
│   ├── classifier.py   — 分类（A/B/C级别、类别、相关性）
│   ├── analyzer.py     — 深度分析（策略要点、因子描述）
│   ├── reporter.py     — 保存结果、批量摘要、钉钉推送
│   └── notifier.py     — 钉钉通知
├── utils/wx_fetcher.py — 直接从微信抓取内容
├── results/            — 分析结果（summary.md, report.md, *.py）
├── logs/               — 日志
├── .processed.json     — 已处理文章/笔记ID（xhs_前缀区分来源）
└── .xhs_subscriptions.json — XHS订阅列表（用户+关键词）
```

## 数据流
抓取(微信/XHS) → HTML解析 → 分类(A/B/C) → 深度分析 → 保存结果/推送通知

## we-mp-rss
后台服务(localhost:8001)，管理微信公众号订阅和文章拉取。
需要定期重新登录微信（扫码验证）。

## XHS (小红书)
通过xhs包访问API，需要:
- Sign server (localhost:5005) — Playwright生成x-s/x-t签名
- XHS_COOKIE环境变量 — 浏览器Cookie，定期过期需更新
- 支持: 用户订阅 + 关键词搜索，仅图文笔记
